Module 2.2: Enriching Linear Models

Introduction

This section will focus on three additions to the multiple regression models discussed in the last section:

  • Qualitative predictors

  • Interaction terms

  • Polynomial terms

All three topics are examples of feature engineering: building new input variables from existing ones to capture relationships that a raw linear model would miss. Despite these enrichments, the model remains linear in its coefficients, so everything we have learned so far still applies.

We’ll be using three datasets to demonstrate concepts: the Beerwings dataset, the advertising dataset, and the mtcars dataset which is built into R.

The standard way of incorporating qualitative or categorical variables like Gender into statistical models is via a process called one-hot encoding. If the variable is binary1, we choose one value to serve as the baseline for the model (R does this by alphabetical order so it will choose “F” in this example). The baseline value is recoded with a 0, and the other value with 1. In other words, we would define a new variable \[ GenderM = \begin{cases}1 & \text{if Gender == "Male"} \\ 0 & \text{if Gender == "Female"} \end{cases} \]

and then add this as a predictor in our SLM. We can engineer such variables in R using ifelse:

beerwings$GenderM = ifelse(beerwings$Gender=="M",1,0)
table(beerwings$Gender,beerwings$GenderM)
   
     0  1
  F 15  0
  M  0 15

Then we run lm with the new variable

fit_lm=lm(Beer ~ GenderM,beerwings)

Notice that it adds “M” at the end of Gender to tell us which value is treated as a “1” in the encoding. You can directly use such calls in the future.

For qualitative predictors with \(k \geq 2\) categories, the idea is similar. We choose one category to serve as baseline and then we create \(k-1\) binary variables for the other categories of the form \[ X_j = \begin{cases}1 & \text{if Category ==} j \\ 0 & \text{if Category !=} j\end{cases} \]

Note that != is R language for “not equal to”. The \(X_j\) are called indicator or (group) dummy variables. Again, lm is built to handle this kind of stuff. Let’s illustrate using the Math 37 survey dataset from the Module 1 Assessment

Interaction Terms

Returning to our Beerwings example, suppose we want to jointly look at the impact of Gender and Hotwings on Beer consumption:

beerwings |>
  ggplot(aes(x=Hotwings,y=Beer,color=Gender))+
  geom_point(shape=1)+
  scale_color_manual(values=c("#0072B2","#D55E00"))+
  geom_smooth(method=lm,se=F)+
  theme_classic()
`geom_smooth()` using formula = 'y ~ x'

There are two separate questions we could ask

  1. Do the two lines have the same intercept?
  2. Do the two lines have the same slope?

The first can be answered by considering a multiple regression model of the form

\[ Beer = \beta_0 + \beta_1 \times Gender + \beta_2 \times Hotwings + \varepsilon \]

The \(\beta_1\) serves as the “shift” in intercept between Gender = 0 (F) and Gender = 1 (M) because if Gender = 0 we get

\[ Beer = \beta_0 + \beta_2 \times Hotwings + \varepsilon \] and if Gender = 1, we get

\[ Beer = (\beta_0 + \beta_1) + \beta_2 \times Hotwings + \varepsilon \]

so \(\beta_1\) is the difference in intercepts. This is just another application of multiple regression

fit_lm=lm(Beer~Gender+Hotwings,beerwings)
summary(fit_lm)

Call:
lm(formula = Beer ~ Gender + Hotwings, data = beerwings)

Residuals:
     Min       1Q   Median       3Q      Max 
-16.7854  -5.0496   0.7114   2.7953  18.7114 

Coefficients:
            Estimate Std. Error t value Pr(>|t|)    
(Intercept)   2.5506     3.8125   0.669    0.509    
GenderM      -2.4361     3.3042  -0.737    0.467    
Hotwings      2.0839     0.3512   5.934 2.52e-06 ***
---
Signif. codes:  0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1

Residual standard error: 7.541 on 27 degrees of freedom
Multiple R-squared:  0.6224,    Adjusted R-squared:  0.5945 
F-statistic: 22.26 on 2 and 27 DF,  p-value: 1.946e-06

Note that the coefficient of GenderM was not significant (although interestingly, it was negative!) so we conclude that the intercepts for Male and Female were not significantly different. To test for a difference in slopes, we can add an interaction term and consider the model

\[ Beer = \beta_0 + \beta_1 (Gender) + \beta_2 (Hotwings) + \beta_3 (Gender \times Hotwings)+ \varepsilon \]

To see why this works, note that if Gender = 0, the model becomes \[ Beer = \beta_0 + \beta_2 (Hotwings)+ \varepsilon \] while if Gender = 1, we get \[ Beer = \beta_0 + \beta_1 + (\beta_2+\beta_3) (Hotwings)+ \varepsilon \]

So \(\beta_3\) is the difference in the slopes and \(\beta_1\) is still the difference in the intercepts.

Adding an interaction term is the same as engineering a new variable whose values are \(Hotwings \times Gender\), but again lm will do this automatically if you include both variables with a * between them:

fit_lm_int=lm(Beer~Gender*Hotwings,beerwings)
summary(fit_lm_int)

In this case it is negative, but insignificant so we conclude that gender does not impact beer consumption after accounting for wings consumption (i.e. if you have two people consume the same number of wings, our model would predict the same beer consumption regardless of gender).

We can also include interactions between numeric predictors like in the advertising dataset. Remember that 3D plot we showed in Module 2.1 and my comment about wanting to peal down the surface? Well we can do that if we consider the model \[ sales = \beta_0 + \beta_1 (TV) + \beta_2 (radio) + \beta_3 (TV \times radio) + \varepsilon \]

which creates an estimation function of the form \[ \hat y = a + b_1 x_1 + b_2 x_2 + b_3 x_1x_2 \]

It turns out the interaction between TV and radio is significant:

A second way of visualizing the interactive effect of TV and radio is to compare the slopes of the regression lines for sales vs TV when radio is high vs low. First, we need to engineer a new variable to represent the latter splitting which we will base on the median

mr=median(advertising$radio)
advertising$radio_level=ifelse(advertising$radio>mr,"high","low")
advertising |>
  ggplot(aes(x=TV,y=sales,color=radio_level))+
  geom_point()+
  geom_smooth(method="lm",se=FALSE)+
  scale_color_manual(values=c("darkorange","#17806d"))

You can see that unlike in the beerwings example, the slopes of the regression lines are different for the two groups: when radio spending is high, you also get more “bang for your buck” in terms of TV spending too (higher slope). This is the reason an interactive effect is also sometimes called a synergy effect (at least when the coefficient is positive).

To get a more realistic model which incorporates curvature without the increase, we need to use a data transformation. This and a few other topics related to regression diagnostics will be discussed in 2.3.
